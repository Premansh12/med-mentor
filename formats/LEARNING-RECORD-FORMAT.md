# Learning record format

One file per non-obvious insight, numbered `0001-<dash-case-name>.md`, incrementing. Not a transcript of every lesson — only things worth remembering because they'll shape future teaching decisions.

```markdown
# 0001-confuses-siadh-and-di

## Date
YYYY-MM-DD

## What happened
Brief, factual: which lesson/case surfaced this, what the user got wrong
or got right in a way worth noting.

## The insight
The non-obvious part — not "missed a question about SIADH" but the
underlying pattern, e.g. "reliably confuses SIADH and diabetes insipidus
when a vignette gives urine osmolality without also stating serum sodium
explicitly — needs the two values paired every time until this resolves."

## Implication for future lessons
What this should change: resurface this pairing in 1 week and again in
1 month (spacing); interleave with other sodium-disorder vignettes rather
than drilling SIADH alone (interleaving); flag as a board-day risk if the
exam is close.

## Status
Open / resolved / monitoring — update this line as it evolves rather than
creating a new file for the same underlying issue.
```

Use these records, together with `MISSION.md`, to compute the user's zone of proximal development before each new lesson — see `references/learning-design.md`.
