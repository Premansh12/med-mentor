# RESOURCES.md format

A living list of trusted sources and communities, grouped by type. Add a source the first time it's used to ground a lesson; don't pre-populate speculatively.

```markdown
# Resources

## Primary clinical references
- [Name](url) — what it's good for, e.g. "UpToDate — first-line for
  current treatment algorithms, updated frequently."

## Guidelines / societies
- [Name](url) — issuing body, topic area, year if known
  (e.g. "AHA/ACC 2023 — Chest pain evaluation guideline").

## Question banks / exam prep
- [Name](url) — exam it targets, user's experience with it so far.

## Journals used
- [Name](url) — for what kind of claim (e.g. "NEJM — landmark trial
  citations").

## Communities
- [Name](url or location) — type (subreddit / journal club / society forum /
  local study group), why it fits the user's specialty/stage, and whether
  the user has joined it yet.

## Notes on source hierarchy
If two sources conflict, which one this user's context should default to
(e.g. "for US board prep, prefer board-style question banks over UpToDate's
more nuanced real-practice guidance, since the exam tests the textbook
answer").
```
